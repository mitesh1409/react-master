# Module #2: JavaScript Fundamentals for React

## Optional Chaining (`?.`), Nullish Coalescing (`??`), Logical OR (`||`)

Topics

- Optional Chaining (`?.`)
- Nullish Coalescing (`??`)
- Logical OR (`||`)

---

Here are simple examples of each:

## Optional Chaining (`?.`)

Safely access nested properties without errors if something is `null` or `undefined`:

```javascript
const user = {
  name: "Alice",
  address: {
    city: "Paris"
  }
};

console.log(user.address?.city);        // "Paris"
console.log(user.phone?.number);        // undefined (no error!)
console.log(user.settings?.theme);      // undefined

// Without optional chaining, this would throw an error:
// console.log(user.phone.number);      // ‚ùå Error: Cannot read property 'number' of undefined
```

## Nullish Coalescing (`??`)

Provides a default value only when the left side is `null` or `undefined`:

```javascript
let username = null;
console.log(username ?? "Guest");       // "Guest"

let count = 0;
console.log(count ?? 10);               // 0 (because 0 is not null/undefined)

let message = "";
console.log(message ?? "No message");   // "" (empty string is not null/undefined)
```

## Logical OR (`||`)

Provides a default value when the left side is any falsy value (`false`, `0`, `""`, `null`, `undefined`, `NaN`):

```javascript
let username = null;
console.log(username || "Guest");       // "Guest"

let count = 0;
console.log(count || 10);               // 10 (0 is falsy)

let message = "";
console.log(message || "No message");   // "No message" (empty string is falsy)
```

**Key difference between `??` and `||`:**  
Use `??` when you want `0` or `""` to be valid values,  
and use `||` when you want to treat them as "missing" and use the default instead.
